월 노출 기준

다음 중 하나라도 선택한 월에 해당하는 티켓을 표시합니다. (챕터 대시보드와 동일)

  • ① 처리완료일
  • ② 기한
  • ③ 티켓 생성일
  • ④ 상태 카테고리 변경일 — 상태 카테고리(할 일 / 진행 중 / 완료)가 바뀐 날짜 기준.
  • ⑤ 티켓 수정일 — 마지막 변경 발생일 (2026-06-09 일괄편집 건은 제외).
사용법
  • ‹ › 로 월 이동, '이번 달'로 현재 월 복귀.
  • 상태 카드 클릭 → 해당 상태만 필터.
  • 기본은 최근 업데이트순. 생성일 · 기한/완료일 · 업데이트일 헤더 클릭으로 정렬 (3회 클릭 시 정렬 해제).
  • 담당자 · 유형 칩과 검색으로 필터. 완료·보류 숨기기 토글, 별(★) 버튼으로 즐겨찾기.
  • 현재 필터·월은 URL에 반영 (?who= &status= &type= &q= &month=).
  • 데이터 10분 캐싱. 하단 '강제 새로고침'으로 즉시 반영.
상태 · 하위작업
  • 상태 : Jira 상태 카테고리(완료 / 진행 중 / 해야 할 일) 기준 자동 분류. 배포 대기·이슈 아님만 별도 표기.
  • 상태 카드·상단 건수 : 작업 티켓 + 하위작업(BE·FE·QA·Plan) 전체 기준 (에픽·이니셔티브 제외).
  • 하위작업(BE·FE·QA·Plan) : 이름 앞 ↳ 표시 + 옅은 배경 + 유형 배지로 구분. 일반 작업 티켓엔 배지 없음.
