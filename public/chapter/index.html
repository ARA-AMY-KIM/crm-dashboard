대상 티켓
  • 프로젝트 : CRM / PNB / CON
  • 담당자 : 김아라 · 이동희 · 이성주 · 장은주 · 최세영
  • 티켓 타입 : 작업 티켓 + Plan 티켓 포함
노출 기준

다음 중 하나라도 선택한 월에 해당하는 티켓을 표시합니다.

  • ① 처리완료일
  • ② 기한
  • ③ 티켓 생성일
  • ④ 상태 변경일 — 티켓 상태의 카테고리(할 일 / 진행 중 / 완료)가 바뀐 날짜 기준입니다. 같은 카테고리 내 상태 변경(예: 기획 진행 중 → 디자인 대기)은 반영되지 않습니다.
  • ⑤ 수정일 — 담당자 변경, 코멘트, 상태 변경 등 티켓에 마지막으로 변경이 발생한 날짜입니다.

※ 작업 티켓과 Plan 티켓 중 하나라도 해당 월에 포함되면 세트로 함께 표시됩니다.

Plan 티켓
  • 작업 티켓 바로 아래 들여쓰기로 표시되며, 구분 열에 점(●) + Plan 텍스트로 구분됩니다.
  • 상태 필터와 카운트는 작업 티켓 기준으로만 적용됩니다. Plan 티켓은 작업 티켓에 세트로 따라옵니다.
  • Plan 티켓 담당자가 5인 내이면 검색 및 담당자 필터에 동일하게 적용됩니다.
  • 상위 작업 담당자가 5인 외일 경우, 해당 작업 티켓은 목록에 노출되지만 필터·카운트에는 포함되지 않으며 흐리게 표시됩니다.
  • 하나의 작업 티켓에 여러 Plan 티켓이 연결될 수 있습니다.
  • 테이블 상단 'Plan 티켓 같이 보기' 토글로 표시 여부를 선택할 수 있습니다.
상태 구분 — 작업 티켓
  • 완료 : 최종 완료
  • 진행 중 : 기획·디자인·개발·QA 진행 중 / 대기 포함
  • 배포 대기 : 배포 대기
  • 할 일 : 시작 전
  • 이슈 아님 : 보류 / 이슈 아님
상태 구분 — Plan 티켓
  • 기획 완료 : 기획 완료 (초록)
  • 진행 중 : 요구사항 분석 / 기획 진행 중
  • 할 일 : 할 일
  • 이슈 아님 : 이슈 아님 / 작업 중지
연결된 티켓

PNB·CON 티켓에 implements 관계로 연결된 CRM 티켓은 별도 행으로 노출되지 않고, 해당 티켓의 '연결된 티켓' 열에만 표시됩니다.

즐겨찾기
  • 행 오른쪽 끝 별 아이콘(☆)을 클릭하면 즐겨찾기로 등록됩니다. 작업 티켓과 Plan 티켓 모두 즐겨찾기 가능합니다.
  • 즐겨찾기는 내 브라우저에만 저장됩니다. 별도 로그인이 필요 없습니다.
  • 브라우저 캐시를 지우거나 다른 브라우저·기기에서는 보이지 않습니다.
  • 같은 티켓은 월이 달라도 즐겨찾기 상태가 유지됩니다.
  • 테이블 상단 '내 즐겨찾기' 버튼으로 즐겨찾기된 티켓만 필터링할 수 있습니다.
기타
  • 상단 ‹ › 버튼으로 월 이동 가능, '이번 달' 버튼으로 현재 월로 즉시 이동
  • 프로젝트 / 담당자 / 상태 필터 및 키워드 검색 지원 (티켓명 · 담당자 · 티켓 키)
  • 기한/완료일 · 업데이트일 기준 정렬 가능 (헤더 클릭, 3회 클릭 시 초기화)
  • 데이터는 10분 캐싱 적용 (하단 강제 새로고침으로 즉시 반영 가능)
참고사항
  • Jira와 자동 연동되므로 별도 작업 없이 항상 최신 데이터가 반영됩니다. 기존 컨플루언스 문서는 그대로 사용하셔도 됩니다.
  • 작업 티켓과 Plan 티켓의 날짜가 다를 경우, 동일한 세트가 여러 달에 걸쳐 노출될 수 있습니다.
